Аналитический обзор проделанной работы по кейс-задаче № 1
 
1. Функциональность
Программа корректно обрабатывать входные данные и выдает правильные результаты:
• корректно и последовательно выдает запрос для пользователя для ввода даты рождения (день, месяц, год);
• определяет день недели на основе модуля datetime;
• проверяет високосный ли год рождения пользователя и выводит на экран;
• делает расчет возраста пользователя, учитывая день рождения в текущем году;
• формирует дату рождения, используя стилизованный вывод цифр в виде символов (LED Scoreboard).
2. Производительность
Программа производит все вычисления «мгновенно» благодаря отсутствию сложной обработки данных. Программа не обращается к внешним ресурсам, что способствует быстроте вычислений. Минимальная сложность функций обеспечивает высокую производительность. Что дает возможность использования программы на маломощных устройствах.
3. Удобство использования
Взаимодействия с пользователем реализовано в консольном формате, что обусловлено заданием. Программа выдает последовательные запросы на ввод данных. При вводе некорректных данных, программа выдает сообщения об ошибках. Вывод данных после работы программы хорошо считаемый с учетом использования в консольном режиме.
4. Безопасность
Безопасность реализована на уровне, который соответствует заданию. Обеспечена защита от некорректного ввода за счёт конструкций (try/except). 
5. Масштабируемость
Программа написана в логике построения с помощью отдельных функции для каждой подзадачи. Программа позволяет дальнейшее масштабирование. Она легко допускает добавление новых функций без необходимости переписывать основ логики.
6. Сопровождаемость
Подробные комментарии описывают работу каждого компонента. Такая структура облегчает понимание сути и позволяет быстро разобраться в логике программы.
7. Переносимость
Программа написана на Python и не используется внешних зависимостей, что дополнительно повышает переносимость решения.
8. Качество кода
Код написан в соответствии со стандартом:
• разделён на функции;
• читабелен и подробно комментирован;
• названия переменных и функций логически отражают их назначение;
• использованы встроенные возможности языка.
9. Тестирование
Программа протестирована на входных данных:
• корректные даты;
• некорректные даты;
• нечисловые вводы.
Программа обрабатывает ошибочный ввод и выводит сообщение об ошибке.
10. Вывод
Программа по кейс-задаче № 1 соответствует требованиям задания:
• реализован последовательный ввод даты рождения пользователя;
• все функции программы реализованы согласно заданию;
• реализован вывод результата штатной работы программы в консольном режиме.
• реализован вывод сообщений об ошибке при некорректном вводе данных;
